## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check existing issues to avoid duplicates.

When creating a bug report, include:

- **Clear title and description**
- **Steps to reproduce** the behavior
- **Expected behavior**
- **Actual behavior**
- **Screenshots** if applicable
- **Environment details:**
  - OS: [e.g., macOS 14.0]
  - Node version: [e.g., 18.17.0]
  - npm version: [e.g., 9.6.7]
  - mcp-fortress version: [e.g., 0.1.0]

**Use this template:**

```markdown
**Describe the bug**
A clear description of what the bug is.

**To Reproduce**
1. Run command '...'
2. Click on '....'
3. See error

**Expected behavior**
What you expected to happen.

**Screenshots**
If applicable, add screenshots.

**Environment:**
 - OS: [e.g., macOS]
 - Node: [e.g., 18.17.0]
 - mcp-fortress: [e.g., 0.1.0]

**Additional context**
Any other context about the problem.
```

### Suggesting Enhancements

Enhancement suggestions are welcome! Please include:

- **Clear title and description**
- **Use case** - Why would this be useful?
- **Alternatives considered**
- **Additional context** or mockups if applicable

**Use this template:**

```markdown
**Is your feature request related to a problem?**
A clear description of the problem.

**Describe the solution you'd like**
What you want to happen.

**Describe alternatives you've considered**
Other solutions you've thought about.

**Additional context**
Mockups, examples, or references.
```

### Questions

Have a question? Please:

1. Check the [documentation](https://github.com/mcp-fortress/mcp-fortress#documentation)
2. Search [existing issues](https://github.com/mcp-fortress/mcp-fortress/issues)
3. Open a [discussion](https://github.com/mcp-fortress/mcp-fortress/discussions)

## Development Setup

> **Note:** The source code is currently closed. This section is for potential future contributors once the project becomes open source.

If the project becomes open source in the future:

```bash
# Clone the repository
git clone https://github.com/mcp-fortress/mcp-fortress.git
cd mcp-fortress

# Install dependencies
npm install

# Start development server
npm run dev
```

## Pull Request Process

> **Note:** Pull requests are not currently accepted as the source code is closed.

If the project becomes open source:

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request
